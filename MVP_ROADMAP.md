# SRS Flashcard App - MVP開発ロードマップ

## MVP（Minimum Viable Product）定義

**目標**: 基本的なSRS機能を持つ、複数ユーザーが使える学習アプリを2ヶ月以内にリリース

**コアバリュー**:
- ユーザーは効率的に暗記学習ができる
- FSRS v4アルゴリズムによる科学的な復習スケジューリング
- シンプルで使いやすいUI

## 機能優先順位マトリックス

### Must Have（MVPに必須）
| 機能 | 重要度 | 複雑度 | Phase |
|------|--------|--------|-------|
| ユーザー登録・ログイン | 高 | 低 | 1 |
| デッキ作成・編集・削除 | 高 | 低 | 2 |
| カード作成・編集・削除 | 高 | 中 | 3 |
| 基本カード（表・裏） | 高 | 低 | 3 |
| FSRS v4アルゴリズム | 高 | 高 | 4 |
| 学習セッション | 高 | 中 | 5 |
| 4段階評価 | 高 | 中 | 5 |
| ダッシュボード | 中 | 低 | 6 |

### Should Have（MVPに含めたい）
| 機能 | 重要度 | 複雑度 | Phase |
|------|--------|--------|-------|
| 画像添付 | 中 | 中 | 3 |
| カード検索 | 中 | 低 | 3 |
| キーボードショートカット | 中 | 低 | 5 |
| レスポンシブデザイン | 中 | 中 | 全体 |
| プロフィール編集 | 中 | 低 | 1 |

### Could Have（MVP後に追加）
| 機能 | 重要度 | 複雑度 | 実装時期 |
|------|--------|--------|----------|
| タグ機能 | 中 | 中 | Phase 2 |
| 学習統計グラフ | 中 | 高 | Phase 2 |
| インポート/エクスポート | 低 | 中 | Phase 2 |
| 穴埋めカード | 中 | 高 | Phase 3 |
| 音声添付 | 低 | 中 | Phase 3 |
| デッキ共有 | 中 | 高 | Phase 4 |

### Won't Have（MVP・Phase 2には含めない）
| 機能 | 理由 |
|------|------|
| ソーシャル機能 | スコープ外 |
| モバイルアプリ | Web版が安定してから |
| AI生成カード | 技術的複雑度が高い |
| チーム機能 | まずは個人利用を完成させる |

## リスク管理

### 高リスク項目
1. **FSRS v4の実装複雑性**
   - 対策: 早期にプロトタイプを作成し、動作確認
   - 代替案: SM-2アルゴリズムへのフォールバック

2. **画像アップロードのパフォーマンス**
   - 対策: 画像サイズ制限と圧縮
   - 代替案: 画像なしでのMVPリリース

3. **スケジュールの遅延**
   - 対策: 2週間ごとのマイルストーン確認
   - 代替案: Could Have機能の削減
